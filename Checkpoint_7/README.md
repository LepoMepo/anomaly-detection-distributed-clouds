## LogBERT

Блы реализован LogBERT подход для детекции аномалий. Был создана трансформерная архитектура на чистом pytorch `bert_pytorch/model`. 

#### 1. Датасет

В качестве датасета использовался [HDFS](https://github.com/logpai/loghub/tree/master/HDFS) датасет. 

#### 2. Предобработка 

Для шаблонизации логов использовался [Drain3](https://github.com/logpai/Drain3). Каждая строка лога получала свой id шаблона, затем логи формировались в последовательности по `block_id`.

Датасеты делятся на следующие:
- train: только нормальные последовательности;
- validation_normal: только нормальные последовательности;
- validation_abnormal: только аномальные последовательности;
- test_normal: только нормальные последовательности;
- test_abnormal: только аномальные последовательности.

#### 3. Модель

LogBERT модель решает задачу masked log key prediction (MLKP), т.е. предсказание замаскированного шаблона логов в последовательности. Для формирования батчей использовался динамический паддинг. 

Архитектура: embedding+positional embedding → encoder → linear → logits

Обучение: CrossEntropyLoss по замаскированным токенам.

#### 4. Скоринг аномалий

Для каждого блока считаем:

```
anomaly_ratio = (число неверно предсказанных токенов / число замаскированных токенов) усредненное за 5 раз
```

В данном случае используются разные маски для скрытия разных токенов, чтобы избежать случайности при выборе маскируемых токенов.

Далее аномалия определяется, если `anomaly_ratio > threshold`.

### 5. Подбор порога

Порог определялся по максимуму F1 метрики на валидации.

#### 6. Оценка модели

Модель оценивалась по Precision/Recall/F1 метрикам по каждому блоку.

#### 7. Результаты

Полученные метрики на тесте при подобранном пороге `threshold=0.0`:
- `precesion =  0.98`
- `recall = 0.808`
- `f1 = 0.885`

Использованные гиперпараметры:
- `src_vocab_size=24`, `tgt_vocab_size=24`
- `src_seq_len=512`, `tgt_seq_len=512` 
- `d_model=512`, `N=6`, `h=8`
- `dropout=0.1`, `d_ff=2048`
- `epochs=16`, `mask_ratio=0.65`
- `top-k=3`

Результаты работы представлены в ноутбуке `logbert.ipynb`.

## MLflow

Был реализова MLflow сервис, который развернут на VPS сервере. Для развертывания сервиса необходимо изменить `.env` файл. Для запуска сервиса необходимо указать команду:

```
docker compose up -d
```

MLflow будет доступен на порту 5050, хранилище артефактов будет доступно на порту 9001.

Пример работы с сервисом показан в ноутбуке `logbert.ipynb`.
